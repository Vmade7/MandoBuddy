# MandoBuddy UI Requirements Document (ui_REQ.md)

## 1. 文档概述
本文档为 MandoBuddy 4天MVP版本提供完整的UI/UX设计规范与前端实现要求。文档严格对齐《MB设计文档》核心目标，聚焦轻量化、高体验、零学习成本的网页端交互设计，确保前端团队可在极短周期内高效交付可用、好看、易维护的界面。

## 2. 设计原则
- 极简启动：用户从打开页面到进入学习不超过三次点击，无冗余弹窗与强制注册拦截。
- 视觉降噪：界面留白充足，信息层级清晰，英文操作指令与中文学习内容严格区分排版区域。
- 即时反馈：每一次点击、播放、答题必有视觉与微动效响应，练习结果0.5秒内呈现。
- 情感化陪伴：用鼓励文案、进度徽章、平滑过渡降低零基础用户的畏难情绪，营造轻松学习氛围。
- 前端优先：所有交互依赖浏览器原生能力与本地存储，不依赖复杂后端渲染，保障弱网与离线体验。

## 3. 视觉规范
**色彩系统**
- 主行动色：暖橙色 #FF6B35（用于核心按钮、主进度条、高亮CTA）
- 辅助品牌色：清新蓝 #4ECDC4（用于次要按钮、链接、文化模块标识、图标）
- 全局背景：浅灰 #F8F9FA（降低长时间学习的视觉疲劳）
- 文本颜色：深灰 #2D3748（主内容）、中灰 #718096（辅助说明/占位符）
- 状态颜色：成功绿 #48BB78（正确/完成/收藏）、警示红 #F56565（错误/提示/删除）

**字体与排版**
- 英文与界面字体：-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif
- 中文内容字体："Noto Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif
- 字号阶梯：页面标题 24px，模块标题 20px，正文 16px，注释与按钮 14px
- 行高设置：英文 1.5，中文 1.6，段落间距 12px
- 字重规范：标题 SemiBold (600)，正文 Regular (400)，按钮 Medium (500)

**间距与圆角**
- 基础计量单位：8px（所有内边距、外边距、组件高度均按8的倍数递增）
- 卡片圆角：12px，按钮圆角：8px，输入框与标签圆角：6px
- 内容区最大宽度：桌面端 1200px，两侧留白自适应

## 4. 核心页面与组件设计
**首页（Landing Page）**
- 顶部导航：Logo（左侧） + 简洁菜单（Learn, Culture, Profile, Help）
- 英雄区域：核心价值标语 + 主按钮“Try a Lesson” + 次按钮“Sign Up Free” + 3张课程预览卡片（悬停微抬升）
- 底部页脚：版权信息、帮助链接、反馈邮箱，背景使用浅灰分割线

**学习详情页（Lesson Page）**
- 顶部固定栏：面包屑导航 + 全局进度条（填充动画） + 返回按钮
- 左侧内容区：图文讲解模块 + 自定义音频播放器 + 中英对照例句（中文加粗，英文浅灰）
- 右侧交互区：随堂练习卡片（单选/填空） + 发音跟读区（录制/回放按钮） + 收藏星标
- 底部悬浮区：固定“Next Lesson”按钮，练习完成前置灰，完成后变为主色并附带滑入提示
- 自动流转逻辑：当前模块完成后平滑滚动至下一练习区，无需手动翻页

**个人中心（Profile Page）**
- 数据看板：环形进度图（CSS绘制） + 已完成课程数 + 连续学习天数徽章
- 标签切换栏：我的收藏 / 错题本，点击切换内容列表
- 列表项设计：每条包含标题、关联课程、复习按钮、删除图标，支持左滑或长按删除（移动端）
- 设置区块：退出登录、清除本地数据（点击触发二次确认弹窗，防止误操作）

**文化卡片页（Culture Page）**
- 网格布局：桌面三列、平板两列、手机单列
- 卡片内容：封面图 + 标题 + 简短英文介绍 + “学相关词汇”入口
- 详情页：左图右文排版，底部附带可发音词汇标签，点击高亮并播放音频

**通用组件规范**
- 音频播放器：播放/暂停/进度条/音量控制，播放时显示动态波形占位，支持0.8x/1.0x/1.2x倍速
- 录音组件：点击开始录制显示红色呼吸光效，时长计数器实时更新，录制完成后提供试听与重录双按钮
- 反馈弹窗：成功/错误/提示统一采用顶部居中滑入样式，3秒自动消失，支持手动关闭
- 徽章组件：解锁时触发缩放至110% + 金色光晕扩散动效，伴随轻微震动（移动端支持时）

## 5. 交互与动效规范
- 按钮点击反馈：按下缩放至95%，松开恢复，时长0.1秒，背景色明度微降
- 页面切换动效：淡入淡出（Fade）配合轻微上滑10px，时长0.3秒，缓动函数 ease-out
- 练习结果反馈：答对时卡片边框泛绿 + 显示鼓励文案；答错时正确选项标红 + 下方展开解析区，整体过渡0.4秒
- 本地保存提示：数据同步时右下角显示“✓ Saved”微Toast，透明度100%至0%渐变，不阻断操作
- 滚动行为：模块切换使用平滑滚动（scroll-behavior: smooth），滚动时长0.4秒，避免跳跃感
- 音频加载状态：未加载完成时显示骨架屏占位，播放按钮置灰，加载完毕自动恢复可点击状态

## 6. 响应式与适配要求
**断点划分**
- 桌面端：≥1024px
- 平板端：768px - 1023px
- 手机端：≤767px

**布局策略**
- 桌面：左右分栏结构，学习内容占60%，交互练习占40%，两侧固定边距
- 平板：上下堆叠结构，内容区全宽，练习区紧随其后，导航收缩为图标+文字组合
- 手机：单列流式布局，顶部导航转为底部标签栏（首页/学习/个人），练习区卡片化，录音按钮固定底部安全区

**触控优化**
- 所有可点击元素最小触控区域 44x44px
- 滑动操作支持横向翻页（课程卡片、文化卡片），带惯性滚动与回弹边界
- 长按录音改为轻点切换录制状态，避免移动端误触

## 7. 无障碍与可访问性
- 色彩对比度：文本与背景对比度严格 ≥ 4.5:1，符合 WCAG 2.1 AA 标准
- 键盘导航：全局支持 Tab 键顺序遍历，聚焦状态使用 2px 蓝色外发光清晰标识
- 语义化结构：图片与音频提供 alt/title 属性，表单控件绑定 label，交互元素补充 aria-label
- 字体缩放：支持浏览器默认字体缩放至200%时不破坏布局与重叠
- 动效降级：尊重系统“减少动效”偏好（prefers-reduced-motion），自动关闭非必要动画

## 8. 交付物与前端对接要求
**设计交付**
- Figma 源文件：包含完整页面流、组件库、设计令牌（Design Tokens）变量
- 交互原型链接：提供主要流程的可点击演示（首页→课程→练习→反馈→进度保存）
- 资源导出：SVG 图标优先，图片统一压缩为 WebP 格式，音频提供 MP3 320kbps 标准文件

**前端实现规范**
- 样式管理：使用 CSS 自定义属性（:root）集中管理颜色、字号、间距、圆角，便于后期主题切换
- 动效实现：统一使用 CSS transition / keyframes，避免 JS 强制重排与布局抖动
- 状态映射：localStorage 数据结构需与UI组件状态严格对应（如 progress 对象映射进度条，favorites 数组映射收藏列表）
- 性能约束：首屏资源总大小 ≤ 1.5MB，音频文件按需懒加载，字体仅加载 Regular 与 SemiBold 字重

**验收标准**
- Lighthouse 性能评分 ≥ 90，可访问性评分 ≥ 95
- 多端视觉一致性误差 ≤ 2px，核心交互无卡顿或掉帧
- 所有P0功能在主流浏览器（Chrome, Safari, Firefox, Edge）最新版中表现一致
- 本地存储断点续学成功率 100%，页面刷新后状态无损恢复

## 9. MVP边界说明（4天范围）
- 本期不包含AI发音评分界面，跟读模块仅提供标准音频对比与录音回放功能
- 不涉及多语言界面切换，全站操作文案固定为英文
- 不涉及复杂用户权限与后台管理UI，仅保留基础注册登录与个人中心展示
- 文化模块与课程模块内容以静态JSON注入，UI仅负责渲染与交互绑定

本规范作为前端开发与UI走查的唯一基准，所有界面实现需严格遵循上述视觉、交互与性能要求，确保4天内交付一个体验完整、运行稳定、具备验证价值的MVP产品。
