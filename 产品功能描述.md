# 文档一：MandoBuddy 产品功能描述文档（MVP版）

## 1. 项目概述
MandoBuddy 是一款面向海外零基础/初级中文学习者的轻量化网页学习平台。产品以“低门槛、即时反馈、文化轻融合”为核心，采用纯前端优先架构，用户无需下载即可通过浏览器3次点击开启学习。本期 MVP（4天交付周期）聚焦核心学习闭环与交互体验验证，并在发音练习模块引入**前端 AI 评分交互流**作为微创新，快速验证“即时纠音”对初学者留存的价值。

## 2. 目标用户与核心场景
| 用户类型        | 核心诉求                           | 典型使用场景                                  |
| --------------- | ---------------------------------- | --------------------------------------------- |
| 零基础成人      | 流程极简、发音有人指导、碎片化学习 | 通勤时段手机浏览器打开，学10分钟拼音+短句     |
| 海外青少年/华裔 | 趣味互动、拒绝枯燥背诵、文化共鸣   | 周末平板共学，完成发音对比小游戏解锁文化卡片  |
| 短期来华人士    | 场景实用、即学即用、发音准确       | 出差前一周突击高频短句，跟读对比后导出复习    |
| 文化爱好者      | 语言+文化融合、轻松探索            | 晚间浏览节日/美食卡片，同步学习关联词汇与发音 |

## 3. MVP 核心功能矩阵
### 3.1 前端核心体验
- **响应式布局**：一套代码适配 Desktop / Tablet / Mobile，核心操作区触控目标 ≥44×44px
- **流畅动效**：按钮点击缩放、页面淡入滑动、进度条填充动画均通过 CSS/JS 实现，帧率稳定 ≥50fps
- **本地存储机制**：学习进度、收藏词汇、错题记录使用 `localStorage` 持久化，断网/刷新不丢失

### 3.2 核心学习模块
- **拼音学习**：声母/韵母/声调图文拆解 + 预录标准音频 + 跟读按钮
- **词汇学习**：场景卡片（图片+音频+英文释义+例句）+ 收藏功能
- **句子学习**：短对话音频 + 中英双语字幕 + 逐句循环播放
- **文化彩蛋**：轻量图文卡片（节日/礼仪/冷知识）+ 关联词汇跳转

### 3.3 轻量反馈与进度
- **随堂测验**：选择题/填空题即时反馈 + 简单英文解析
- **错题本**：自动收录错误题目，支持按模块筛选与一键重练
- **进度追踪**：可视化进度条 + 完成徽章激励，学完自动平滑滚动至下一模块

## 4. AI 微创新：前端发音评测交互流
### 4.1 功能定位
在 MVP 阶段不依赖重型后端，通过前端原生能力构建完整的**“听-录-评-反馈”交互闭环**，验证即时发音指导对零基础用户的正向激励作用。

### 4.2 用户操作流程
1. **听标准音**：点击播放预录音频，同步显示动态波形动画
2. **录制跟读**：长按/点击录音按钮，麦克风实时采集并显示呼吸光效与音量指示
3. **智能评分**：点击“分析”，触发 1.5 秒加载动画，前端生成评分结果
4. **反馈引导**：显示环形进度条分数 + 英文鼓励文案 + 声调提示（如 `Try lowering your pitch on the 3rd tone 📉`）
5. **对比与重试**：支持标准音与录音交替播放，可一键重录或进入下一题

### 4.3 MVP 实现策略
- **音频采集**：`MediaRecorder` + `Web Audio API` 实时波形渲染
- **评分逻辑**：MVP 采用音频时长/能量特征模拟评分，预留 `Azure Speech SDK` 替换接口
- **状态管理**：`recording → analyzing → scored` 三态机，避免界面卡顿
- **降级方案**：若浏览器不支持，自动切换至“纯录音回放对比”模式

## 5. 信息架构与核心交互路径
```
首页（Home）
  ↓ 3次点击原则
学习入口（Learn）→ 选择课程 → 课程详情页
  ↓ 学完自动流转
练习区（Quiz / AI Pronunciation）→ 即时反馈 → 解锁下一节
  ↓ 数据沉淀
个人中心（Profile）→ 进度看板 / 收藏 / 错题本 / 设置
```
- **无阻断设计**：完成当前模块后自动平滑滚动至练习区，无需手动翻页
- **双语分离**：操作界面纯英文指令，学习内容中英对照，降低认知负荷

# 文档二：MandoBuddy 专业背景信息文档（技术/架构/交付向）

## 1. 市场痛点与产品定位
### 1.1 行业现状
- 现有中文学习产品多偏向应试（HSK导向）或重型 APP，安装包大、注册流程长
- 零基础海外用户面临“语法晦涩、发音无人纠正、学习路径复杂”三大门槛
- 网页端轻量入门工具存在市场空白，缺乏“即开即学+即时反馈”的完整闭环

### 1.2 MandoBuddy 定位
以 **Frontend-First Architecture** 为核心，打造低摩擦、高愉悦感的中文入门体验。通过剥离重后端依赖、聚焦前端交互与本地化数据管理，实现 MVP 快速验证，为后续规模化迭代积累真实用户行为数据。

## 2. 技术架构与选型依据
| 层级           | 技术选型                                          | 选型依据                                                     |
| -------------- | ------------------------------------------------- | ------------------------------------------------------------ |
| **前端框架**   | React 18 + Vite 5 + Tailwind CSS                  | 热更新快、打包体积小、响应式开发效率高，契合4天迭代周期      |
| **状态与存储** | `useState`/`Context` + `localStorage`             | 避免冗余状态库，本地存储保障离线可用，降低服务器成本         |
| **后端服务**   | FastAPI + Supabase（MVP轻量）                     | 异步高性能、自动 Swagger 文档、免费层满足 MVP 用户量与 Auth 需求 |
| **部署方案**   | Vercel（前端） + Docker（后端）                   | 零配置 CI/CD、全球 CDN 加速、容器化保证环境一致性            |
| **音频与AI**   | `MediaRecorder` + `Web Audio API` + Mock 评分引擎 | 浏览器原生支持、零外部依赖、演示零翻车，预留真实 AI 接口     |

## 3. AI 发音评测的渐进式实现方案
### 3.1 MVP 阶段：前端交互验证
- **核心目标**：跑通“采集→可视化→评分反馈”完整交互流，验证 UX 价值
- **技术路径**：
  1. 用户录音生成 `Blob`
  2. 前端提取基础音频特征（时长、平均能量、静音段比例）
  3. 通过轻量算法映射至 60-95 分区间，匹配预设声调提示文案
  4. 返回结果驱动 UI 状态切换（`analyzing → scored`）
- **优势**：开发周期 ≤ 2 天，无需 API Key，演示稳定，符合学术交付“重验证轻生产”原则

### 3.2 V2 阶段：真实 AI 集成
- **目标方案**：接入 `Azure Cognitive Services Speech SDK`（浏览器版）
- **架构调整**：前端直连 → 后端代理转发（防 Key 暴露 + 跨域处理 + 请求限流）
- **评估维度**：准确度（Accuracy）、流利度（Fluency）、完整性（Completeness）、声调匹配度
- **数据回流**：评分结果存入 Supabase，用于后续个性化复习推荐

## 4. 数据流与状态管理设计
```
用户操作 → 前端捕获事件 → 本地状态更新 → localStorage 持久化
                              ↓
                      注册/登录触发同步
                              ↓
              FastAPI JWT 验证 → Supabase 读写 → 返回最新状态
                              ↓
              前端合并本地与云端数据（时间戳冲突解决）
```
- **冲突处理**：以最新操作时间戳为准，避免多端覆盖
- **降级策略**：网络异常时优先保障本地写入，恢复后自动排队同步
- **安全边界**：敏感操作（清除数据、退出登录）强制二次确认；音频文件不上传云端（MVP期）

## 5. 敏捷开发策略与风险控制
| 风险项            | 影响程度 | 应对策略                                                     |
| ----------------- | -------- | ------------------------------------------------------------ |
| AI 评分准确性不足 | 中       | 明确 MVP 定位为“交互验证”，评分文案采用鼓励型话术，避免绝对化表述 |
| 浏览器兼容性差异  | 低       | 首页添加提示 `For best experience, use Chrome 90+`；提供降级开关 |
| 4天周期交付压力   | 高       | 严格 P0 优先级控制，P1 功能留空占位；每日站会同步阻塞点      |
| 演示环境翻车      | 高       | 预录 15 秒核心流程视频嵌入 PPT 备用；准备 Mock 开关一键切换  |
